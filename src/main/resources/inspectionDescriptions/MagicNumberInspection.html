Suggests to avoid magic numbers on code.

Magic numbers are numeric literals used on code that could be referenced in a lot of places but are not constants, then are hard to understand and maintain.

It will always report for number literals in return statements, switch cases and too conditionals (except on testing counting by zero).

Additional reporting options:

  • Report for multiplier: check if a magic number is used as multiplier (excluding -1).
  • Report for properties: check if a magic number is used as default value on properties (excluding 0).
  • Report for parameters: check if a magic number is used as default value on parameters (excluding 0).
  • Report for arguments: check if a magic number is used as argument (excluding 0 and the default parameter value).
